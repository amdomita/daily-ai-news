# 每日AI与科技资讯 - 2026-05-07

## AI/人工智能

- **Meta Muse Spark 正式发布，但放弃开源路线** — Meta 于4月8日发布 Muse Spark，这是 Alexandr Wang 领导的超级智能实验室推出的首个多模态推理模型，支持工具调用、可视化思维链与多智能体编排，但与以往 Llama 系列不同，Muse Spark 完全闭源、仅云端可用，引发社区争议。（TechCrunch / The New Stack）

- **Google Gemma 4 开源发布，31B 模型性能超越 400B 量级竞品** — Google DeepMind 于4月2日发布 Gemma 4 系列，包含 E2B、E4B、26B MoE 及 31B Dense 四款模型，采用 Apache 2.0 协议，支持256K上下文、视觉与音频多模态、140+语言，代理基准 tau2-bench 得分较上代提升13倍。（Google DeepMind / Hugging Face Blog）

- **Google Gemini 3.1 Ultra 发布，原生支持200万Token上下文** — Gemini 3.1 Ultra 是 Google 年度最重要模型发布，具备200万 Token 上下文窗口，原生跨文本、图像、音频、视频多模态推理，无需转录中间步骤，并内置沙箱代码执行工具。（Google）

- **Anthropic 启动 Project Glasswing，向企业开放 Claude Mythos Preview** — Anthropic 向 AWS、Apple、Google、JPMorgan Chase 等机构开放其未公开前沿模型 Claude Mythos Preview 的受控访问权，用于发现并修复关键软件漏洞。（Anthropic）

- **Google 在 ICLR 2026 发布 TurboQuant，大幅压缩 KV Cache 内存开销** — TurboQuant 算法结合 PolarQuant 向量旋转与量化 Johnson-Lindenstrauss 压缩两步流程，显著降低大模型推理中 KV Cache 的内存占用，为长上下文推理提速。（Google Research）

## 开源软件

- **Google 开源多智能体编排测试平台 Scion** — Google 将实验性多智能体协调框架 Scion 正式开源，作为研究和测试复杂代理系统交互行为的标准化平台，面向 AI 应用开发者开放。（InfoQ）

- **Docker 推出 Kanvas，挑战 Helm 与 Kustomize 的 Kubernetes 配置管理地位** — Docker Kanvas 能将 Docker Compose 文件自动转换为 Kubernetes 部署制品，旨在弥合本地开发与云端生产的落差，对 Helm 和 Kustomize 构成直接竞争。（InfoQ）

- **Redis 8.6 发布，带来显著性能提升与内存压缩** — Redis 8.6 在性能和内存利用率上均取得"实质性"改进，同时持续完善 Redis 8 中引入的 Vector Set、GCRA 限速器等 AI 场景特性。（Phoronix）

- **Git 2.54 发布，GitHub Blog 发布详细功能解读** — 开源版本控制系统 Git 更新至 2.54，GitHub 官方博客对新特性与改动进行了深度解析。（GitHub Blog）

## 数据库与中间件

- **Redis 8.6 发布，单命令性能最高提升87%** — Redis 8.x 持续迭代，8.6版本带来"实质性"性能提升与内存占用下降；Redis 8整体相比前代提供最高87%命令加速、2倍吞吐量提升、16倍查询引擎处理能力，并新增 Vector Set 支持 AI 语义检索场景。（Phoronix / Redis Blog）

- **Apache Kafka 4.2.0 于2月发布，4.0.x 持续维护** — Kafka 4.2.0 于2026年2月17日正式发布；4.0.2 作为关键补丁版本于3月发布。Kafka 4.0 是首个完全移除 ZooKeeper 依赖、默认运行 KRaft 模式的大版本，并引入 Share Groups 支持队列语义。（Apache Kafka Blog / Confluent Blog）

- **MySQL 维护 9.x 创新版本轨道，双轨发布策略持续推进** — MySQL 最新稳定版为9.6.0（2025年12月发布），官方维持"创新版"与"长期支持版"双轨并行策略，持续吸纳新特性同时保障企业级稳定性。（VersionLog）

- **KubeBlocks 持续更新，统一管理 MySQL/Redis/Kafka 等数据库** — 云原生数据库管理平台 KubeBlocks 持续发布新版本，支持在 Kubernetes 上统一编排 MySQL、Redis、Kafka、RocketMQ 等主流中间件，降低多数据库运维复杂度。（GitHub / apecloud/kubeblocks）

## 云原生

- **Kubernetes v1.36 "Haru" 于4月22日发布，70项增强稳步落地** — v1.36代号"春"（ハル），包含18项稳定特性、25项进入 Beta、25项进入 Alpha；用户命名空间（User Namespaces）正式 GA，SELinux 卷挂载优化 GA，PSI 压力指标 GA，Mutating Admission Policies GA，为 AI/GPU 工作负载和生产稳定性奠定基础。（Kubernetes Blog）

- **Ingress NGINX 正式宣告退役，社区需迁移替代方案** — 2026年3月24日，Kubernetes SIG Network 与安全响应委员会宣布 Ingress NGINX 停止维护，不再发布任何更新或安全补丁，现有集群需尽快评估迁移至 Gateway API 或其他 Ingress 控制器。（Kubernetes Blog / Diginomica）

- **Clever Cloud 于5月5日将 Kubernetes 1.36 设为默认版本** — 法国云服务商 Clever Cloud 率先将 K8s 1.36 Haru 设为平台默认版本，意味着 v1.36 开始进入主流云平台生产推广阶段。（Clever Cloud）

- **Docker Kanvas 挑战 Helm/Kustomize，简化 Compose 到 K8s 的迁移路径** — Docker 推出 Kanvas 平台，面向中小开发团队提供 Compose→Kubernetes 自动转换能力，降低云原生采用门槛，与现有 Helm Chart 和 Kustomize 形成竞争。（InfoQ）

- **Kubernetes GPU/AI 工作负载支持持续增强，DRA 动态资源分配进入 Beta** — 随 AI 模型训练与推理需求激增，K8s 通过动态资源分配（DRA）Beta 特性为 GPU 等加速硬件提供统一调度接口，`allocatedResourcesStatus` 字段可通过 `kubectl describe pod` 直接查看设备健康状态。（Cloud Native Now / Kubernetes Blog）
